# Get Signoff

Present work for user approval before proceeding.

## Process

### Step 1: Summarize Current State
Gather information about what's ready for review.

### Step 2: Present for Approval

```
## Ready for Signoff

### What Was Done
[Summary of completed work]

### Changes
[List of files modified]

### Key Decisions
[Any important choices made during implementation]

### Questions/Concerns
[Anything that needs user input]

---

Approve to proceed? (yes/no/discuss)
```

### Step 3: Handle Response

**If approved**: Proceed with next phase (commit, PR, etc.)

**If not approved**:
- Ask what needs to change
- Make adjustments
- Present again

**If discuss**:
- Address concerns
- Clarify decisions
- Iterate until resolved
